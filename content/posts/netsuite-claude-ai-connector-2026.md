בסוף המאמר תמצאו מדריך שלב שלב איך לבצע את החיבור בפועל

גרסה 1 של NetSuite לשנת 2026 מביאה התקדמות מרחיקת לכת ביכולות פלטפורמה מונחות-AI, שירותי REST משופרים, ותכונות הרחבה חדשות בכל מחסנית SuiteCloud - שמעצימות הן מפתחים והן משתמשים עסקיים. תמצאו אינטגרציות מוכנות לייצור עם מערכות AI מסחריות, כמו Claude ו-ChatGPT, דרך Model Context Protocol (MCP), וסיוע מבוסס-AI שמאיץ משימות פיתוח SuiteScript. בנוסף, גרסה 2026.1 סוגרת פערים בשירותי REST web services של SuiteTalk, ומכשירה את הקרקע לפרישת אינטגרציות מדור ישן מבוססות SOAP.

## AI תופס את מרכז הבמה עם שלוש יכולות פורצות דרך

### שירות ה-NetSuite AI Connector: אינטראקציות בשפה טבעית עם NetSuite

NetSuite משתלבת כעת עם פלטפורמות AI מובילות, כולל ChatGPT, Claude, וכל מערכת תואמת MCP, דרך שירות ה-AI Connector החדש. שכבת אינטגרציה ניתנת להרחבה זו מאפשרת לסוכני AI מובילים לגשת בצורה מאובטחת לנתונים וליכולות NetSuite שלכם באמצעות הנחיות בשפה טבעית, תוך ציות מלא למודל האבטחה של NetSuite ולמבנה ההרשאות מבוסס-התפקידים.

![ממשק שפה טבעית של Claude לאינטראקציה עם חשבון NetSuite](/images/blog/netsuite-claude-ai-connector-2026-1.jpeg)

המחבר חושף קבוצה עשירה של כלים דרך SuiteApp של כלי MCP הסטנדרטיים. מאחר שהאינטגרציה עוקבת אחר סטנדרטים פתוחים, מפתחים יכולים לתמוך בכל פלטפורמת סוכן תואמת MCP - תוך הסתמכות על שכבת האבטחה המוכחת של SuiteCloud.

**מה זה מאפשר:** אפליקציות AI גנרטיביות כמו Claude ו-ChatGPT הן כעת ממשקים ישירים עם פונקציונליות ERP. מערכות AI יכולות לבקש פעולות NetSuite, כגון אחזור נתונים, פעולות עסקאות, ויצירת רשומות, ישירות - בעוד שותפי SuiteCloud Developer Network (SDN) יכולים לבנות ולמנות מכירה של SuiteApps מבוססי-AI המתחברים ליכולת החדשה הזו ללא יצירת אינטגרציות משלהם.

### SuiteCloud Developer Assistant: פיתוח מבוסס-AI ב-VS Code ו-Cline

SuiteCloud Developer Assistant מציג עוזר קידוד מונחה-AI המשתלב עם Visual Studio Code דרך תוסף Cline. מפתחים יכולים לייצר קוד SuiteScript 2.1, כולל בדיקות יחידה, באמצעות הנחיות בשפה טבעית. העוזר גם מייצר אובייקטים של SuiteCloud Development Framework; מסביר או מבצע refactor לקוד קיים; ועוקב אחר תקני SuiteScript API, הגדרות TypeScript, וסכימות SDF.

![יצירת בקשת בדיקת יחידה SuiteScript ב-VS Code עם תוסף Cline](/images/blog/netsuite-claude-ai-connector-2026-2.png)

**היתרונות מיידיים:** מפתחים חדשים יוכלו לספק קוד איכותי מהר יותר הודות לעקומת למידה קצרה עם SuiteScript, בעוד שוותיקים ימצאו קל יותר לתחזק ולהרחיב SuiteApps בקנה מידה ארגוני.

### שיפורי SDF מחזקים את ניהול מחזור החיים

**הוק לפני הסרת התקנה:** מפתחי SuiteApp יכולים כעת לבצע לוגיקה מותאמת אישית מיד לפני הסרת התקנת אפליקציה. על ידי הגדרת `sdf-installation-script` בקובץ `application.xml`, מפתחים יכולים לאוטמט משימות ניקוי, כגון הסרת רשומות תלויות, קבצים, ונתונים, כדי למנוע נכסים עזובים לאחר הסרת התקנה. זה מבטיח תהליך הסרת התקנה נקי יותר, מפחית עומס תמיכה ומספק חוויית לקוח משופרת.

**הרשאות קבצים עבור קבצי .ss ו-.ssp:** SDF תומך כעת בהרשאות קבצים מפורשות עבור קבצי `.ss` ו-`.ssp` באמצעות תג XML לאישור, וסוגר פער שבו הרשאות שנקבעו בממשק המשתמש לא נשמרו במלואן בפרויקטי SDF. מפתחים מקבלים שליטה מלאה יותר, ביקורת אבטחה טובה יותר, וזרימות עבודה CI/CD משופרות כעת שהרשאות משתמש ומערכת נעקבות בבקרת מקור.

**סוג סקריפט כלי מותאם:** סוג סקריפט הכלי המותאם החדש מאפשר למפתחים לבנות כלים שלקוחות AI חיצוניים יכולים להפעיל דרך שירות ה-NetSuite AI Connector. סקריפטים אלה גם תומכים במודול N/log וכוללים תמיכה בממשק משתמש לרישום וניהול כלים פרוסים דרך MCP.

## REST APIs מגיעים לשוויון מלא עם שירותי SOAP Web Services

עדיפות עליונה של NetSuite בגרסת 2026.1 היא סגירת כל פערי REST API הנותרים לתמיכה במעבר חלק מ-SuiteTalk SOAP. שיפורים אלה מיישרים את שירותי ה-REST web services של SuiteTalk ואת ה-APIs עם בנצ'מרקים בתעשייה ומאפשרים למפתחים להתרחק מפתרונות אינטגרציה מבוססי פרוטוקול SOAP מדור ישן.

- **פעולות אצווה הומוגניות:** מפתחים יכולים כעת להגיש מספר פעולות מאותו סוג בקריאת REST אחת, דבר שאידיאלי לייבוא מאסיבי או עדכונים המוניים. פעולות אצווה יפעלו באופן אסינכרוני, מפחיתות תקורת רשת תוך שיפור תפוקה וביצועים.

- **פעולת יצירת טופס:** פעולת אתחול-סגנון חדשה זו מאפשרת למפתחים לאכלס מראש שדות טופס. דוגמה היא יצירת חשבונית מהזמנת מכירה - כעת משתמשים צריכים להוסיף רק מידע ספציפי נדרש. לאחר מכן הם יכולים להגיש את הרשומה הסופית ללא יצירה מחדש ידנית של לוגיקת ממשק המשתמש.

- **פעולות צירוף והפרדה:** REST APIs תומכים כעת בצירוף והפרדה של רשומות או קבצים, מה שמקל על משימות נפוצות כגון ניהול מסמכים, זרימות עבודה מותאמות אישית, ולוגיקת אינטגרציה.

- **פעולת SelectOptions:** מפתחים יכולים כעת לאחזר את האפשרויות הזמינות לשדות מסוימים ישירות דרך REST, ומפשטים מיפוי בתוך אינטגרציות שותפים, פלטפורמות iPaaS, ואפליקציות מותאמות אישית.

## איך מחברים Claude ל-NetSuite – שלב אחר שלב

### דרישות מוקדמות

- חשבון **Claude Pro** (לפחות) – לא עובד בחינם
- גישת **Administrator** ל-NetSuite להגדרה ראשונית

### שלב 1: התקנת SuiteApp ב-NetSuite

נכנסים ל-NetSuite ומנווטים ל-**SuiteApp Marketplace**, מחפשים **"MCP Tools"**, בוחרים את ה-**MCP Standard Tools SuiteApp** ולוחצים **Install**.

### שלב 2: יצירת תפקיד (Role) מיוחד

מסיבות אבטחה, כלי MCP לא יכולים להתחבר מ-Claude ל-NetSuite תחת תפקיד Administrator. יש ליצור תפקיד ייעודי לצורך זה ולהקצות אותו למשתמש שיבצע את החיבור.

נתיב: **Setup → Users/Roles → Manage Roles** → יוצרים תפקיד חדש עם ההרשאות המינימליות הנדרשות.

### שלב 3: חיבור Claude לחשבון NetSuite

שתי דרכים אפשריות:

**דרך א' – הקונקטור הרשמי (הפשוטה יותר):** נכנסים ל-claude.ai, הולכים ל-**Search and tools ← Add connectors**, ובקטגוריית Connectors בלשונית Web בוחרים את **NetSuite AI Connector**.

**דרך ב' – חיבור ידני עם URL:** בחשבון Claude Pro, לוחצים על שם החשבון בפינה שמאל למטה ובוחרים **Settings → Connectors → Add custom connector**. בשדה ה-URL מזינים:

```
https://<companyid>.suitetalk.api.netsuite.com/services/mcp/v1/all
```

מחליפים את `companyid` במזהה החברה שלכם מה-URL של NetSuite.

### שלב 4: אישור ה-OAuth ב-NetSuite

תפתח דף NetSuite שיבקש לבחור תפקיד – בוחרים את **תפקיד ה-MCP** שיצרתם (לא Administrator). מופיע מסך שמבקש לאשר את החיבור. לאחר לחיצה על **Allow**, מופיעה הודעת הצלחה "Successfully Connected".

### שלב 5: הגדרת הרשאות הכלים

ליד ה-NetSuite MCP app, לוחצים על **Configure** ולכל הכלים המוצגים בוחרים **"Allow unsupervised"** בתפריט הנפתח.

![הגדרת הרשאות הכלים ב-Claude](/images/blog/netsuite-claude-ai-connector-2026-3.png)

### שלב 6: התחלת עבודה

ה-AI יפרש את הבקשות שלכם וייצא את פעולות NetSuite המתאימות. פעולות שיוצרות או משנות נתונים ידרשו אישור לפני ביצוע.

![סשן עבודה עם Claude ו-NetSuite](/images/blog/netsuite-claude-ai-connector-2026-4.png)

**דוגמאות לשאלות שאפשר לשאול מיד:**
- "סכם את 10 הלקוחות המובילים לפי הכנסה"
- "נתח את הוצאות הספקים לפי קטגוריה"
- "מהו מצב התזרים הנוכחי של החברה?"

## נקודות חשובות לדעת

- נדרש לפחות **Claude Pro** כדי להוסיף את הקונקטור.
- MCP מוגבל כברירת מחדל ויש להפעילו בצורה מפורשת.
- הכלים פועלים רק עם הרשאות המשתמש שביצע חיבור – לא מעבר לכך.
- לאחר ההתקנה, נוצר רשומת אינטגרציה אוטומטית ב-NetSuite הנגישה דרך **Setup → Integration → Manage Integrations**.

שירות NetSuite AI Connector הוא שירות אינטגרציה מונחה-פרוטוקול התומך ב-Model Context Protocol (MCP) שמעניק ללקוחות דרך גמישה וניתנת להרחבה לחיבור ה-AI שלהם ל-NetSuite. עם תמיכה ב-bring-your-own-assistant ועיצוב אגנוסטי-פרוטוקול ניתן להרחבה, אתם אינם כלואים לספק AI יחיד.
