התקדמות ה-AI משנה את פני המקצוע הפיננסי בקצב חסר תקדים. אנו עוברים במהירות מעידן של "השלמת קוד" (Code Completion) פשוטה לעבודה עם סוכנים אוטונומיים (Agentic AI) המסוגלים לתכנן ולבצע משימות מורכבות. אולם, כארכיטקטים של מערכות, עלינו להכיר ב"קיר" הטכני: **עומס הקשר (Context Bloat)**. לא ניתן להזין את כל מתודולוגיית הביקורת, ספרי החשבונות והנהלים הארגוניים לתוך חלון ההקשר מבלי לגרום לבלבול של המודל או לבזבוז מיותר של טוקנים.

כאן נכנס לתמונה ה-Finance Transformation. בהקשר האג'נטיבי, מדובר במעבר של איש הפיננסים מתפקיד של **מעבד נתונים (Data Processor)** לתפקיד של **אורקסטרטור של מערכות AI (AI Orchestrator)**. המפתח לשינוי זה הוא ניהול "ידע מבוזר" (Distributed Knowledge) באמצעות Agent Skills — פתרון המאפשר "טעינה עצלנית" (Lazy Loading) של הקשר עמוק וממוקד רק כאשר הסוכן מזהה צורך בכך בתהליך ה-Plan שלו.

## הבנת הקונספט: מהו Agent Skill?

על פי התקן המוצג ב-agentskills.io, "כישור" אינו רק הנחיה, אלא **יחידת יכולת** המאפשרת Collocation (מיקום משותף) של לוגיקה, סקריפטים ונכסים. זהו הממשק שדרכו אנו משפיעים על סגנון ואופי הפלט של הסוכן.

### המבנה הטכני של Skill

כדי שהסוכן יוכל "לגלות" (Discover) את הכישור, עלינו להקפיד על מבנה קשיח:

- **תיקייה ייעודית**: שם התיקייה מגדיר את מרחב השמות של הכישור.
- **קובץ `skill.md`**: לב הכישור. זהו קובץ Markdown הכולל את ההנחיות.
- **YAML Front Matter**: זהו ה-Interface של המודל לשלב ה-Discovery. ללא תיאור מדויק ב-YAML, הסוכן לא ידע מתי "לטעון" את הכישור.
- **Resources**: סקריפטים (כגון Python) או קבצי נתונים שנמצאים באותה תיקייה.

### השוואת כישורים באקו-סיסטם הפיננסי

| מאפיין | Global Skills (כישורים גלובליים) | Project Specific Skills (ספציפיים לפרויקט) |
|--------|----------------------------------|---------------------------------------------|
| מטרה | יכולות רוחביות (SQL, Pandas, ניתוח דוחות) | לוגיקה עסקית ספציפית, נהלי ביקורת פנימיים |
| דוגמה | ניתוח סטטיסטי של סדרות עתיות | התאמה למבנה ה-API של מערכת ה-ERP הארגונית |
| אחסון | ספריית ה-Global של הסוכן | תיקיית `.agent/skills/` בתוך ה-Repository |

## שלב 1: הקמת תשתית הפרויקט וסביבת העבודה

כדי להבטיח שהסוכן יוכל לבצע "אורקסטרציה" לידע שסיפקנו, עלינו להגדיר היררכיית תיקיות מסודרת. בתוך סביבת העבודה (למשל ב-Anti-gravity), ניצור את התשתית הבאה:

**מבנה התיקיות:** `.agent/skills/finance-audit/`

**יצירת קובץ ה-`skill.md` הראשוני** — ה-YAML Front Matter הוא קריטי. הוא משמש כ"מצפן" עבור ה-LLM בזמן תכנון המשימה:

```yaml
---
name: finance_audit_anomaly_detector
description: Expert skill for identifying financial irregularities, including round-sum transactions, weekend entries, and accounting anomalies in journal files.
---
# הנחיות לזיהוי חריגות פיננסיות

השתמש בכישור זה כאשר המשתמש מבקש לסרוק יומני תנועות (Journal Entries).
עליך להפעיל את סקריפט ה-Python המצורף (`detect_anomalies.py`) כדי להפיק ממצאים כמותיים.
```

## שלב 2: פיתוח סקיל לזיהוי חריגות (Anomaly Detection)

הכוח האמיתי של ה-Skill הוא היכולת להריץ לוגיקה קשיחה. במקום לבקש מהסוכן "לחפש דברים מוזרים", אנו מספקים לו סקריפט Python מובנה בתוך ה-Skill. זה משפר את הדיוק האודיטורי ומונע **הזיות (Hallucinations)**.

**פרומפט מערכת ליצירת הלוגיקה (System Prompt):**

> "פעל כארכיטקט מערכות פיננסיות. צור סקריפט Python לזיהוי חריגות המשתמש ב-Pandas. הסקריפט חייב לזהות: 1. תנועות בסופי שבוע. 2. תנועות בסכומים עגולים (כפולות של 1,000 או 10,000). 3. חריגות סטטיסטיות מעל 3 סטיות תקן."

**הקוד המוטמע בתוך ה-Skill (`detect_anomalies.py`):**

```python
import pandas as pd
import numpy as np

def analyze_journal(file_path):
    df = pd.read_csv(file_path)
    df['date'] = pd.to_datetime(df['date'])

    # 1. זיהוי תנועות בסופי שבוע (5=Saturday, 6=Sunday)
    weekend_issues = df[df['date'].dt.dayofweek >= 5]

    # 2. זיהוי סכומים עגולים (חשד להערכות ידניות)
    round_amounts = df[df['amount'] % 1000 == 0]

    # 3. זיהוי חריגות סטטיסטיות (Z-Score > 3)
    mean = df['amount'].mean()
    std = df['amount'].std()
    outliers = df[(df['amount'] - mean).abs() > 3 * std]

    return weekend_issues, round_amounts, outliers
```

## שלב 3: בניית סקיל להתאמות בנקים אוטומטיות

התאמות בנקים דורשות טיפול בנתונים מרובי מקורות ובלוגיקה של "התאמה רכה" (Fuzzy Matching).

**פרומפט מפורט להגדרת לוגיקת התאמה (Matching Logic):**

> "כסוכן ביקורת, עליך לבצע התאמה בין קובץ הבנק (`bank_stmt`) לקובץ הספרים (`ledger`). לוגיקת ביצוע:
> 1. בצע התאמה מדויקת על בסיס מספר אסמכתא וסכום.
> 2. עבור תנועות ללא אסמכתא, בצע 'Fuzzy Matching' על בסיס תיאור התנועה וטווח תאריכים של +/- 3 ימים.
> 3. סמן תנועות ללא התאמה כ'פתוחות' וסווג אותן לפי רמת סיכון."

## שלב 4: הפעלה ואימות — הפקת דוח הממצאים

כאשר תבקשו מהסוכן: "בצע ביקורת על קובץ התנועות של רבעון 3", תוכלו לראות ב-Trace שהוא טוען את הכישורים שיצרנו. הסוכן ישתמש ב-`finance_audit_anomaly_detector` כדי להריץ את סקריפט ה-Python ולהפיק פלט מקצועי.

### דוח ממצאים (Financial Audit Findings Memorandum)

**אל:** הנהלת הכספים / ועדת ביקורת
**הנדון:** ממצאי בדיקה אוטומטית — רבעון 3

**1. תמצית מנהלים (Executive Summary)**

סריקה אוטומטית בוצעה על 12,450 תנועות יומן. המערכת זיהתה 42 חריגות הדורשות התייחסות מיידית.

**2. ממצאים מרכזיים (Key Findings)**

- **חריגות ימי מנוחה:** זוהו 12 תנועות שבוצעו בימי שבת/ראשון בסך כולל של 450,000 ש"ח.
- **סכומים עגולים:** זוהו 8 פקודות יומן ידניות בסכומים עגולים (למשל 100,000 ש"ח בדיוק) ללא אסמכתא מצורפת.
- **חריגות סטטיסטיות:** נמצאו 3 תנועות החורגות מ-3 סטיות תקן מהממוצע המחלקתי.

**3. הערכת סיכון (Risk Assessment)**

- **רמת סיכון:** בינונית-גבוהה.
- **משמעות:** חשש לעקיפת בקרות פנימיות ורישום ידני לא מבוקר בסוף תקופה.

**4. המלצות (Recommendations)**

יש לבצע דגימה ידנית של התנועות שסומנו בנספח א' ולדרוש אישורי מנהל רלוונטיים עבורן.

## סיכום והמשכיות: שיתוף ידע וצמיחה ארגונית

המעבר לשימוש ב-Skills הופך את הידע המקצועי של המבקר ל**נכס ארגוני דיגיטלי**. באמצעות כלים כמו `skills.sh`, ניתן להפיץ כישורים אלו בין צוותים, ובכך להבטיח שכל סוכן בארגון פועל לפי אותם סטנדרטים מקצועיים.

### Best Practices לתחזוקת כישורים

- **דיוק ה-YAML:** השקיעו בתיאור הכישור. זהו המפתח ל-Discoverability.
- **עדכניות הלוגיקה:** עדכנו את סקריפטי ה-Python בכל פעם שמתעדכנים תקני החשבונאות או הנהלים הפנימיים.
- **ביזור:** נהלו את הכישורים ב-Git כדי לאפשר גרסאות שונות לכל צוות פיננסי.

באימוץ גישה זו, המבקר המודרני אינו רק מבצע פעולות, אלא הופך ל**ארכיטקט של מערכות חכמות**, המעצב את עתיד עולם הפיננסים.
